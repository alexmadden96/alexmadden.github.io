Network Connectivity Troubleshooting Lab

This lab simulates multiple real-world help desk scenarios involving network connectivity issues. Each case demonstrates how to diagnose, test, and resolve problems using common Windows tools and commands.

Environment

  • Windows 11 virtual machine (client)
  • Host system or router for connectivity tests
  • PowerShell and Command Prompt for diagnostics
  • Event Viewer for log review

Scenario 2: IP Address Conflict

Problem: The workstation randomly loses connectivity and shows timeouts when trying to reach the network gateway or external sites. This points to a possible IP address conflict on the network.

üé• Walkthrough Video

Watch a full walkthrough of the IP conflict troubleshooting below:

Step by Step Diagnosis:

üõ†Ô∏è Test Theory to Confirm the Problem

  • ipconfig /all
  • ping 10.0.2.2 (default gateway)
  • ping google.com
  • arp -a (optional check for conflicting MAC entries)

Conclusion

Pings to the gateway and external hosts show unreachable messages and packet loss, while ipconfig /all reveals a manually assigned IP that matches another device on the network. This confirms an IP address conflict.

Fix the Problem

  • Open the network adapter IPv4 settings and switch IP and DNS back to DHCP (Obtain an IP address automatically).
  • ipconfig /release
  • ipconfig /renew

    • Verify the Solution

  • ipconfig /all (confirm a new, unique IP address is assigned)
  • ping 10.0.2.2
  • ping google.com

Scenario 3: Corrupted Network Stack

Symptom: Shows ‚ÄúConnected, no internet‚Äù with an APIPA (169.254.x.x) address.

Diagnosis:

ipconfig /all
ipconfig /renew

Fix:

netsh winsock reset
netsh int ip reset

Reboot system and verify DHCP lease and connectivity.

Scenario 4: Proxy Misconfiguration

Symptom: Ping tests work, but no websites load in the browser.

Diagnosis: Proxy enabled under Internet Options ‚Üí Connections ‚Üí LAN Settings.

Fix: Disable proxy, refresh browser, confirm access restored.
